IMPLEMENTATION
The Small Office Network Project’s implementation phase requires the physical configuration of the network devices, configuration of static routing, and testing of all components. This phase is by far the most realistic since the planning and designing of the project are put into action. Below is a detailed explanation of the steps involved in the implementation process:
The first step for developing this network include connecting the devices in the network using Cisco Packet Tracer. This includes setting a four routers to connect the networks in each department, layer two switches for the intercommunication in each department, and PCs in each department, in the Server Room, Chairman Room, IT department and the computer department. A server is located in the Server Room to allow services such as file sharing, and name resolution. Ethernet cables are used to connect the PCs and servers to the switches within their departments, while routers apply serial or Ethernet links to connect to one another, making sure the right cable is applied depending on the router interface 
Subsequently the IP addresses and subnets are wit drawn for every department with the understanding that none of the networks will share the same IP address and that all the IP addresses are routable. During the server configuration step, an addressing is done on the server to allow it to have an IP Address so that it can acknowledge other devices. After that, the static routing is set on every router needed so that departments can communicate with other departments they need. Static routes are configured in each router; the similar routes are configured in the other routers to allow the flow of traffic between all departments. Checking is done using the show ip route on each router as shown below to ensure that the routes entered are correct on the routing table.
Last but not the least, directing and checking are done in order to confirm that each network is working as expected. A ping test is done from all the PCs in one department to other PCs in other departments to ensure connectivity is real. If problems occur then special procedures are possible such as: Check routing tables with the DOS command show IP route for errors Verify IP settings for routers (IP address/subnet mask/gateway) Static routes are also checked

METHODOLOGY
The methodology used to accomplish this includes proposed steps to design, implement and simulate a small office network using Cisco Packet Tracer. It starts with network design and topology plot that determines the layout and the necessary devices each department would require, then the IP addressing schemes. Then, devices are set up such as IP addressing and settings of PC’s, servers, routers and switches among them. Static routing is then configured on the routers so that the different departments can communicate with each other. The reachability of the network is demonstrated by ping tests, and the correctness of the created routing tables is checked; if necessary, problems are solved. Lastly, documentation is produced in detailed form so as to descriptive topological network, configurations and tests of the local network in order to fulfill the documentation and correctness of the network. This methodology guarantees that I develop a systematic step by step of constructing a fully functional network as well as helping in familiarizing the person with network configuration and troubleshooting skills..

 NETWORK SEGMENTATION AND ROUTING
On the two noted element of the Small Office Network Project, the network segmentation and routing are the most important elements that will enhance the network reliability, security and management. For example, the network has four subnets: Server Room, Chairman Room, IT Department, and Computer Department; traffic moves from one subnet to another thus minimizing traffic in each subnet. Since these networks are isolated, there is a configuration of static routing on the routers in order to allow for communication between them. For instance, Router 1 controls traffic flow between the Server Room, Chairman Room, and IT Department; Router 2 controls the flow of traffic between the IT Department and the Computer Department. Subdivision of this boosts security, as it restricts each department to access the other, and the flow of traffic in the network is minimal. In summary, network segmentation and routing do not only reduce the complexity and increase the security, performance, availability, butpose to make the network much easier to manage when the organization is expanding.
